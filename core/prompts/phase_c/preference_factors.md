# 决策偏好因子库（Preference Factors）

> 本文件是「圆桌决策透镜」的理论底座。每个偏好因子都包含：因子 ID · 理论依据 · 枚举值 · 用户问题 · 应用举例。
> 主张向量（claim_vector_schema.md）通过 `mapped_factor` 字段引用本文件的因子 ID。
> 张力轴库（tension_axis_library.md）通过因子 ID 把张力两极连接到具体偏好画像。

## 聚类逻辑

以"**偏好作用于决策的哪个环节**"为聚类依据，分为 5 大类：

| 类 | 名称 | 回答的问题 | 作用环节 |
|---|---|---|---|
| A | 风险维度 | 怎么看不确定性 | 面对未知时的态度 |
| B | 时间维度 | 看多远 | 时间视角 |
| C | 目标维度 | 要什么 | 目标取舍 |
| D | 认知维度 | 怎么判断 | 信息处理 |
| E | 行为维度 | 会不会动 | 执行与调整 |

## 类别间关联

- **A × B**：风险偏好与时间偏好高度正相关（保守+长期 vs 激进+短期）
- **A × C**：风险偏好决定目标的上限和下限
- **C × D**：目标决定要看哪些数据
- **D × E**：认知方式决定行动节奏
- **E 是终点**：所有偏好最终通过行为维度落地
- **双向反馈**：E 的执行结果反过来更新 A~D 的偏好参数（强化学习闭环）

## 类 A · 风险维度

### A1 风险厌恶度
- **理论依据**：期望效用理论（von Neumann & Morgenstern, 1944）
- **枚举值**：保守 / 平衡 / 激进
- **用户问题**：面对一个新功能，A 方案稳妥但收益有限，B 方案高风险高回报，你倾向？
- **举例（PM 评审"是否上线 AI 推荐流"）**：
  - 保守型 → 抓"灰度发布/降级方案/回滚机制" → "先 1% 流量灰度，建立指标兜底线"
  - 激进型 → 抓"先发优势/快速试错" → "全量上线，3 天看核心指标决定"

### A2 损失厌恶度
- **理论依据**：前景理论（Kahneman & Tversky, 1979）
- **枚举值**：低 / 中 / 高
- **用户问题**：你更怕"错过一个好功能"还是"做了一个没用的功能"？
- **举例（评审"是否砍掉老功能"）**：
  - 高损失厌恶 → 抓"用户流失数据/替代路径" → "先看用户感知，分阶段下线"
  - 低损失厌恶 → 抓"机会成本/资源聚焦" → "立即下线，集中资源做新功能"

### A3 模糊厌恶度
- **理论依据**：Ellsberg 悖论（1961）
- **枚举值**：低 / 中 / 高
- **用户问题**：方案中关键数据缺失时，你会要求补齐 / 凭经验决策 / 推迟决策？
- **举例（评审"未做调研的需求"）**：
  - 高模糊厌恶 → 抓"用户研究方法/数据验证" → "必须先做 N=20 用户访谈"
  - 低模糊厌恶 → 抓"MVP/快速验证" → "先做最小版本看用户反应"

## 类 B · 时间维度

### B1 时间贴现率
- **理论依据**：跨期决策理论（Samuelson, 1937）/ 双曲贴现（Ainslie, 1975）
- **枚举值**：高（短视）/ 中 / 低（远视）
- **用户问题**：本季度有 100 万收入 vs 半年后有 300 万收入，你选？
- **举例（评审"短期变现 vs 长期价值"）**：
  - 高贴现率 → 抓"GMV/转化率" → 推付费功能
  - 低贴现率 → 抓"留存/LTV/用户心智" → 推增强黏性功能

### B2 耐心阈值
- **理论依据**：跨期决策 + 有限理性（Simon, 1955）
- **枚举值**：低 / 中 / 高
- **用户问题**：一个新功能上线多久没出效果，你会判定失败？
- **举例（评审"是否继续投入数据未起色的功能"）**：
  - 低耐心 → 抓"快速试错/及时止损" → "再给 2 周，否则砍"
  - 高耐心 → 抓"长尾效应/学习曲线" → "用户教育需 3 个月，再观察"

### B3 决策节奏
- **理论依据**：双系统理论（Kahneman, 2011）
- **枚举值**：慢思考 / 平衡 / 快决策
- **用户问题**：评审会上，你倾向当场拍板还是会后再定？
- **举例（评审会面对争议）**：
  - 慢思考 → 抓"决策框架/多方案对比" → "列 3 个方案 RICE 评分，下次会再定"
  - 快决策 → 抓"OKR 优先级/第一性原理" → "基于本季度核心目标，直接选 A"

## 类 C · 目标维度

### C1 目标权重（增长 vs 体验）
- **理论依据**：多准则决策理论（MCDM）
- **枚举值**：增长优先 / 平衡 / 体验优先
- **用户问题**：只能选一个：DAU 涨 10% 但满意度跌 5 分 vs 反过来？
- **举例（评审"加广告位"）**：
  - 增长优先 → 抓"商业化路径/广告效率" → "信息流第 5 位加广告，ARPU+8%"
  - 体验优先 → 抓"用户反感度/体验底线" → "只在低价值场景插广告"

### C2 范围偏好（规模 vs 精专）
- **理论依据**：战略管理理论（Porter）
- **枚举值**：大而全 / 平衡 / 小而精
- **用户问题**：做覆盖广的通用功能，还是服务少数核心用户的深度功能？
- **举例（评审"是否做企业版"）**：
  - 规模派 → 抓"市场扩张/通用功能" → "做通用版覆盖中小企业"
  - 精专派 → 抓"垂直深耕/高 LTV 用户" → "只服务头部 100 家做深定制"

### C3 短期 vs LTV
- **理论依据**：跨期决策 + 客户终身价值理论
- **枚举值**：短期转化 / 平衡 / 长期价值
- **用户问题**：首单 ROI 100% vs 首单 ROI 50% 但 12 月 LTV 高 30%，你选？
- **举例（评审"会员定价"）**：
  - 短期型 → 抓"首购转化/限时优惠" → "前 3 个月低价拉新"
  - LTV 型 → 抓"用户分层/权益设计" → "中等定价 + 强权益绑定"

## 类 D · 认知维度

### D1 数据 vs 直觉倾向
- **理论依据**：双系统理论（Kahneman, 2011）
- **枚举值**：数据派 / 平衡 / 直觉派
- **用户问题**：数据说做 A 但直觉说做 B，你怎么选？
- **举例（评审"是否调整核心交互"）**：
  - 数据派 → 抓"A/B 测试/指标体系" → "先 A/B 测 5% 流量看 7 天"
  - 直觉派 → 抓"用户心智/设计哲学" → "符合直觉路径，直接全量"

### D2 样本量门槛
- **理论依据**：统计显著性 + 可得性启发式（Tversky & Kahneman, 1973）
- **枚举值**：低门槛 / 中 / 高门槛
- **用户问题**：多少样本你才认为"有结论"？
- **举例（评审"基于 50 个访谈的需求"）**：
  - 低门槛 → 抓"早期信号/定性洞察" → "信号够强，可立项"
  - 高门槛 → 抓"统计显著性/量化验证" → "需补 N=200 问卷 + 行为数据"

### D3 信息广度偏好
- **理论依据**：信息搜索成本理论（Stigler, 1961）
- **枚举值**：窄而深 / 平衡 / 广而浅
- **用户问题**：决策前查 1~2 个权威源还是查 10 个对比？
- **举例（评审"竞品对标"）**：
  - 窄深型 → 抓"头部竞品深度拆解" → "深度对比 Top1 实现细节"
  - 广浅型 → 抓"行业全景扫描" → "扫 10 家看共性+差异"

### D4 确认偏误程度
- **理论依据**：认知偏差研究（Wason, 1960）
- **枚举值**：低（找反例）/ 中 / 高（找支持）
- **用户问题**：形成判断后，倾向找证据支持还是主动找反例？
- **举例（评审"PM 已有强偏向的方案"）**：
  - 高确认偏误 → Agent 主动抓"反对意见/失败案例" → "这有 3 个失败案例，请评估"
  - 低确认偏误 → 直接给方案优化建议

## 类 E · 行为维度

### E1 后悔规避度
- **理论依据**：后悔理论（Loomes & Sugden, 1982）
- **枚举值**：低 / 中 / 高
- **用户问题**：决策后多大程度会反复想"如果选另一个会怎样"？
- **举例（评审"两个互斥方案"）**：
  - 高后悔规避 → 抓"可逆决策/分阶段切换" → "选 A 但保留 B 接口，3 月后可切"
  - 低后悔规避 → 抓"All in/破釜沉舟" → "选 A 后全力执行，不留退路"

### E2 沉没成本敏感度
- **理论依据**：沉没成本谬误（Arkes & Blumer, 1985）
- **枚举值**：低（敢断舍离）/ 中 / 高（不甘心）
- **用户问题**：已投入 3 个月效果一般、前景不明，你会？
- **举例（评审"是否砍掉做了半年的功能"）**：
  - 高沉没成本 → Agent 主动抓"机会成本/断舍离框架" → "忘掉过去投入，只看未来 ROI"
  - 低沉没成本 → 直接给砍除路径

### E3 禀赋效应
- **理论依据**：禀赋效应（Thaler, 1980）
- **枚举值**：低 / 中 / 高
- **用户问题**：你对自己提的方案是否比别人提的同等方案更看重？
- **举例（评审"自有方案 vs 团队 challenge 的方案"）**：
  - 高禀赋 → Agent 抓"客观对比框架/第三方视角" → "用 RICE 评分对比客观分数"
  - 低禀赋 → 直接综合各方给最优选

### E4 从众倾向
- **理论依据**：社会比较理论（Festinger, 1954）
- **枚举值**：低（独立）/ 中 / 高（跟随）
- **用户问题**：团队多数选 A 但你倾向 B，坚持还是跟随？
- **举例（评审会多数意见与 PM 倾向不一致）**：
  - 高从众 → 抓"独立判断框架/逆向思维" → "提供 3 个独立判断维度帮你评估"
  - 低从众 → 抓"沟通说服/团队对齐" → "教你如何用数据说服团队"

### E5 责任归因风格
- **理论依据**：归因理论（Heider, 1958 / Weiner, 1985）
- **枚举值**：内归因 / 平衡 / 外归因
- **用户问题**：结果不达预期，你倾向归因于自己的判断还是外部环境？
- **举例（评审"上线后不达预期的复盘"）**：
  - 内归因 → 抓"决策反思/个人方法论" → "重新审视当时的决策依据"
  - 外归因 → 抓"市场分析/环境变量" → "评估市场变化对决策的影响"

## 拓展规范

新增因子时：
1. 选定所属类（A~E），分配下一个序号 ID（如 A4、F1 新类）
2. 补全 5 要素：理论依据 / 枚举值 / 用户问题 / 举例
3. 在"因子 ID 速查表"加一行，标注张力轴默认归属
4. 若引入新张力轴，同步更新 `tension_axis_library.md`
