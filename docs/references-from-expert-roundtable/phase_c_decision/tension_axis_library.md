# 张力轴库（Tension Axis Library）

> 本文件枚举「圆桌决策透镜」可识别的【决策张力轴】。每条轴 = 一对对立的决策方向，两极各自挂靠 `preference_factors.md` 的偏好因子。
> 主张向量（`claim_vector_schema.md`）通过 `axis` 字段引用本库的轴 ID，通过 `pole` 字段定位在两极中的哪一极。
> 核心作用：让"张力两端"不是凭空命名，而是落在一条有理论支撑、可挂靠因子的标准轴上。

---

## 一、张力轴的结构

每条轴包含：
- `轴 ID`：轴的唯一名称（中文短语，如 `转化-体验`）
- `+ 极 / − 极`：两端的语义
- `两极挂靠因子`：每一极绑定的决策偏好因子 ID（引用 `preference_factors.md`）
- `典型对撞`：在 PM/投放场景下，哪两类角色主张会落在这条轴的两极
- `画像名`：两极极端化后的决策偏好画像名（用于 Step 5 的 🅰/🅑 命名）

> 张力 = 同一条轴上，两条主张分别落在 `+` 极与 `−` 极 → 形成对撞。

## 三、轴的选取与主导张力判定

当多条主张分布在多条轴上时，主持人按 `facilitator_prompt.md` Step 3 的三标准选主导张力：

1. **冲突强度**：两极立场差距越大，张力越主导（如一方要全量、一方要 1% 灰度）。
2. **角色权重**：该轴上对撞双方的 `weight` 越高，越主导（议题责任方的对撞优先）。
3. **议题相关度**：哪条轴直指议题核心（评审"加广告位"→ `转化-体验` 轴天然最相关）。

## 五、拓展规范

新增张力轴时：
1. 命名为 `X-Y` 形式的对立短语，确保两极语义清晰对立。
2. 补全两极语义 + 两极挂靠因子（必须引用 `preference_factors.md` 已有因子；若无对应因子，先在因子库新增）。
3. 给出典型对撞角色 + 两极画像名。
4. 在"轴枚举表"加一段，并在"因子 → 轴 反查"表补全归属。
5. 若新轴依赖新因子，先按 `preference_factors.md` 的拓展规范新增因子，再回填本库。
