# JuryPersonas HTML 报告设计约束

本文件是报告渲染器的内置执行约束说明。HTML 报告视觉参考为 Linear 风格,采用 Linear 深色信息面板风格;`reporting/reference-design.md` 仅保留历史 SpaceX-inspired 视觉资产参考,不得覆盖本文件。

## 设计目标

1. 场景适配:内部可由 `scenario` 和实际组合的 `mode` 决定结构,但对外必须转成业务可读名称。
2. HTML 优先:本地最终产物优先生成 `.html`,Markdown 作为飞书/纯文本 fallback。
3. 用户视角:默认报告必须面向使用 Skill 的业务用户,不得把 DAG、mode、bundle、mock responder、raw JSON 等工程术语作为主内容展示。
4. 行动在前:第一屏先展示行动优先级;分析依据、用户反馈和评分解释作为支撑判断。
5. 决策分层:共识、分歧、张力、双路径、待办分别呈现,不把主持人总结写成裁决。
6. 边界显性:没有真实反馈数据、没有 MSI-Net 权重、使用 mock reaction 等限制必须在报告中可见。

## 视觉风格

- 参考:以 Linear 风格(深色信息面板、克制配色、等宽数字)为视觉基准。
- 字体:系统无衬线栈,中文优先;数字、mode badge、代码与日志使用等宽栈。
- 页面宽度:最大约 920px,居中,左右留白 24px;内容密度服务决策阅读,不做营销页大屏 hero。
- 背景:Linear 深色基底 `#08090a`,面板使用 `#141517` / `#191b1e`,细线使用 `#232629` / `#1c1e21`。
- 强调方式:主强调色为 `#6e79d6`,只用于焦点、关键数字、当前状态;语义色只对应风险/警告/正向状态。
- 标题:Hero 约 30-42px,区块标题约 21px,标题使用轻微负字距,中文行高不得低于 1.15。
- 层级:用面板嵌套、闭合细线、极弱 radial-gradient 表达层级;禁止单侧描边和大面积撞色块。
- 表格:优先表达结构化信息;表头使用深色面板和小字号,数据值保持易扫读。
- 标签:默认展示业务可读标签,如评审类型、目标人群、参与角色数、视频时长、平台;mode/scenario 只能放入折叠技术附录。
- 图标:禁用 emoji;需要图标时使用 inline SVG。
- 证据:禁止大段 JSON 直接常开铺满页面;长 JSON 使用可折叠区和最大高度。
- 图片:HCI 标注图、热力图等本地产物必须可在 HTML 中直接预览。

## 内容规则

1. 不出现“结论先行”“一句话总结”这类字面标题;直接把判断写在段落开头。
2. 不预测转化率、点击率、完播率等绝对值。
3. HCI 指标只陈述预测性客观观察,不替代真实眼动。
4. `mode/aggregate-distribution-gap` 没有真实反馈数据时,只说分布差异,不说真实效果。
5. 使用 mock reaction 时必须用用户可理解的话声明其为本地回归样例,不要在主报告中裸露 `mock_llm_responder`。
6. 短视频报告必须说明分析依据:使用了哪些关键帧、口播、截图或转写;没有完整视频解码时必须明确说明。
7. 评分区必须解释 0-10 分值区间和每个指标代表的含义,不能只展示均值。

## 场景布局

| 场景 | HTML 布局 |
|---|---|
| 短视频 | 行动优先级 → 陪审结果 → 时间轴/关键帧观察 |
| PRD | 行动优先级 → 张力/双路径状态 → 需求结构与决策风险 |
| 设计稿/单界面 | 行动优先级 → HCI/结构观察 → 可理解性与体验断点 |
| 详情页/商品卡 | 行动优先级 → 价格/权益/规则 → 信任证据与转化阻力 |
| 营销文案 | 行动优先级 → 文案结构 → claim/evidence/CTA/risk |
| 含 distribution-gap | 增配/降配人群表必须进入证据区 |
| 含 heatmap/cross-page/annotate | HCI 观察区必须前置 |
