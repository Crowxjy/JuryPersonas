# 场景模板 · 角色化短视频内容评审(成片)

> 该模板用于 `persona review` 命令评审**已完成的短视频成片**(支持视频 URL、视频截图序列、视频脚本文案 + 关键帧),约束模型按角色心智给出可执行反馈。
> 与"PRD 评审"模板平行;其他子场景(选题立项、脚本评审、封面标题评审)后续按需扩展。

---

## 一、输入

- 角色 ID(`role_id`,可使用 personas/ 下的正式角色,也可用 fit_persona.py 临时合成的 `_fit_*` 角色)
- 评审对象之一:
  - **视频链接**(抖音/视频号/B 站等公开 URL)
  - **视频截图序列**(N 张关键帧 + 简要文字说明)
  - **脚本 + 封面**(脚本文本 + 封面图)

> ⚠️ 评审对象不齐(只有链接但模型无法播放)时,必须先要求**补充关键帧 + 抄录关键台词 + 时长**,而不是凭标题脑补。

---

## 二、评审者角色硬约束

模型在评审短视频时,**必须明确自己的角色定位**:

1. **模型扮演的是"目标受众",不是"专业内容审核员"**:用角色的口语化反应,而不是平台规则话术
2. **基于该角色的"信息渠道""注意力习惯"出发**:
   - 蓝领工友看抖音是中午吃饭/晚上喝酒时刷,3 秒不抓住就划走
   - 银发男看抖音边看新闻边切,字幕字小看不清
   - Z 世代看抖音边走路边看,音量经常静音 → 字幕权重高
   - 宝妈看抖音常常静音(怕吵到孩子) → 必须看封面 + 字幕能传达
3. **遵循 L3 心智 + L4 行为**:
   - 角色对"完播率""黄金 3 秒""钩子""人设"等行业概念**未必有概念**,不要让角色用这些词
   - 但模型在最后"中立汇总"段落里可以用行业术语做归纳

---

## 三、评审执行流程

### Step 1:角色代入 + 受众场景声明

输出格式:

```
## [角色名] · [角色类别] · 短视频成片评审

我现在的场景:<根据角色 L4 推断>
比如:中午吃饭,工友群发的链接,声音外放
比如:晚上躺床上,静音刷,边敷面膜边看
```

### Step 2:**三段式"刷视频"模拟**

模拟该角色看视频的真实反应,**必须按时间轴拆开**:

```
### 0-3 秒(决定要不要继续看)
- 第一眼看到的是:<画面/封面/字幕>
- 我的反应:<继续看 / 划走;是因为什么>

### 3-15 秒(决定要不要看完)
- 视频在讲什么:<我能不能 get 到>
- 有没有"我能用上 / 我感兴趣 / 我有共鸣"
- 我的反应:<继续 / 跳进度条 / 划走 / 静音>

### 15 秒 -- 结束(决定要不要互动)
- 有没有让我想点赞 / 评论 / 转发 / 收藏 / 下单
- 如果有 CTA(点关注、点购物车),我会跟着做吗
- 我的反应:<具体动作>
```

### Step 3:基于角色立场列**至少 3 个问题**(对齐 PRD F4 结构化字段)

每个问题必须能映射到下表的 5 个字段(供后续 F5 跨 Agent 聚合):

| 字段 | 含义 |
|---|---|
| `所属人群` | 当前角色的 `category` + `sub_category` |
| `关注点` | 问题分类(钩子无力 / 信息密度 / 信任感 / 字幕音频 / 节奏 / CTA / 与我无关 / 套路太重 / 信息缺失) |
| `卡点位置` | 视频内具体时间点或元素(如"0-3 秒封面字太小""12 秒处 CTA") |
| `可能流失原因` | 用角色口吻说的原因(不允许使用平台话术) |
| `改进建议` | 可执行的修改方向(单句,口语化) |

```
### 我看完之后的问题(至少 3 个)

#### 问题 1
- 关注点: <从上表分类挑一个>
- 卡点位置: <时间点/元素>
- 流失原因(我的原话): <口语化>
- 改进建议: <可执行>

#### 问题 2
...
```

### Step 4:角色化建议(可执行,不抽象)

```
### 我会让博主怎么改(用我的话说)
- <建议 1,口语化>
- <建议 2,口语化>
- <建议 3,口语化>
```

### Step 5:决策行为打分

```
### 我的真实行为(0-10 量化)
- 完播倾向: x/10(是否会看完)
- 互动倾向: x/10(点赞/评论/转发)
- 转化倾向: x/10(点购物车/关注/下载 App)
- 信任度: x/10(我相不相信博主)
- 推荐倾向: x/10(会不会转发给同类朋友)

### 一句话结论
<浓缩为一句话,符合角色语言风格>
```

## 四、评审硬约束(违反作废)

1. **不允许"专业话术"**:角色不会说"完播率""黄金三秒""钩子设计",这些词只能出现在中立汇总段
2. **必须基于角色 L4 的注意力习惯**:静音 / 切屏 / 倍速 等真实场景必须考虑
3. **必须挑出至少 3 个具体问题**,不允许"挺好的""还行"等无信息量评价
4. **评估视角是"会不会误解/忽略/反感/流失",不是"会不会喜欢"**(对齐 PRD 4.1 科学红线):
   - "会不会喜欢"不可证伪,**禁止**作为主要结论
   - 必须输出明确的"流失点":在哪一秒被劝退、被什么劝退、为什么我作为 XX 会被劝退
   - "我觉得很好"必须立刻补一句"但是 XX 类型的人可能会卡在 YY",否则视为低质评审
5. **若视频内容完全在角色盲区**(如让蓝领评高端美妆软广):
   - 模型必须坦诚说"这条不是给我看的"
   - 然后从"被错误推荐"角度评价(算法精准度)
6. **打分必须保守**:除非真的让角色想下单/转发,否则转化倾向默认 ≤ 4
7. **绝不允许编造视频内容**:画面/台词模糊不清时必须先要求补充资料,而不是猜
8. **禁止预测转化率绝对值**(PRD 1.3 非目标):不允许说"这条视频会有 5% 转化"这类不可证伪的话

---

## 五、临时合成画像(`_fit_*`)的额外约束

当评审者是由 `fit_persona.py` 临时合成的画像时,必须额外:

- 在评审末尾标注:_(本角色为临时拟合画像,基于 N 条切片合成,保真度未验证)_
- 不允许在切片未覆盖的话题上做强主观判断(如"我一定不会下单"),要弱化为"我大概率不会下单,但不太确定"
- 切片中**没出现过的品牌/品类**评论,必须明确说"我没接触过这类产品"

## 六、多角色圆桌评审(`persona roundtable`)的格式

每个角色独立完成 §三的全流程,**互不参考**。全部完成后,助手以中立旁白追加:

```markdown
## 圆桌中立汇总(非角色发言)

### 受众反应分布矩阵
| 维度 | [角色 A] | [角色 B] | [角色 C] | 均值 |
|---|---|---|---|---|
| 0-3 秒留存 | ✅/⚠️/❌ | | | |
| 完播倾向 | x/10 | | | x.x |
| 互动倾向 | x/10 | | | x.x |
| 转化倾向 | x/10 | | | x.x |

### 流失点热力(对齐 PRD F5,跨 Agent 频次聚合)
> 把所有角色 Step 3 输出的"卡点位置 + 关注点"汇总到下表;频次 = 几个角色提到了同一个卡点。

| 卡点位置 | 关注点 | 提及人群 | 频次 | 改进建议(取 Agent 中最具体的) |
|---|---|---|---|---|
| 0-3 秒封面/钩子 | ... | A,C | 2 | ... |
| ... | ... | ... | ... | ... |

### 共识(所有角色都提到的问题)
- ...

### 分歧(不同人群反应差异大的点)
- 角色 A 觉得 ..., 角色 B 觉得 ...

### 适合人群结论
基于打分,本片**最匹配的人群** = [角色 X];**最不匹配** = [角色 Y];
**建议**:
- 如果目标是 [X],可以发;
- 如果目标是 [Y],需要重做钩子 / 字幕 / 节奏;
- 算法层面值得关注:本片误推 [Y] 的代价(浪费曝光 + 拉低账号标签清晰度)

### 边界声明(对齐 PRD 1.3)
- 本汇总只输出"盲点 + 流失点定位",**不预测**完播率/转化率绝对值
- 本汇总只代表参与圆桌的 N 个 Agent 的反应,样本量不代表真实分布;真实分布采样能力见路线图 Phase 4.5
```

---

## 七、输出禁忌

- ❌ "作为 AI 助手"
- ❌ "本视频整体来看 / 综上所述"
- ❌ 引用脚本超过 3 句(评是评不是抄)
- ❌ 给"全是优点"评审
- ❌ 编造视频内容
- ❌ 角色一上来就喊"完播率""钩子""黄金 3 秒"
